# Agent Memory（智能体记忆）

> Agent Memory 不是“聊天记录 + 检索”的别名，而是一个管理过去如何进入现在、并最终把经验固化为能力的分层状态治理系统。

## 核心要点

- **记忆不等于检索**：RAG 解决“资料库里有但模型不知道”的问题，Memory 解决“系统如何在时间中保留偏好、事实变化、教训和约束，并持续影响未来行为”的问题。
- **记忆是分层的**：可按工作记忆、情景记忆、语义记忆、程序性记忆来理解。它们分别对应当下推理、经历回溯、稳定知识压缩和能力固化。
- **记忆有生命周期**：成熟系统至少要处理写入、整理、读取三步，而不是把所有历史一股脑扔进向量库后祈祷召回率拯救世界。
- **高质量记忆必须允许遗忘**：真正困难的不是“怎么存”，而是如何让旧信息失效、退场、可追溯但不继续污染当前行为。
- **记忆的终点是能力**：当经验被反思、验证并沉淀为 Skills 或 SOP，记忆就从“记得”跃迁到了“会做”。

## 详细说明

### 四类记忆

CoALA 把 Agent Memory 放回经典认知架构语境中，可分为四类：

- **工作记忆**：当前真正进入上下文并参与推理的那一小块信息，是所有记忆最终必须通过的瓶颈。
- **情景记忆**：带时间属性的经历记录，例如一次对话、一次工具调用、一次失败尝试，强调“事情是怎么发生的”。
- **语义记忆**：从多次经历中蒸馏出的稳定知识，例如用户偏好、实体关系、长期约束。
- **程序性记忆**：把有效经验压缩成可复用行动模式，例如 Skills、SOP、多步操作套路。

这四类并不是简单的数据标签，而是 Agent 在时间中留下的四种不同残留物。

### RAG 不是 Memory

RAG 的核心动作是“读外部知识并注入当前上下文”，更接近图书馆访问机制；Memory 则必须覆盖写入、更新、替代、失效、删除的完整生命周期，更接近系统对个体和历史的持续理解。

因此，长上下文、聊天历史、向量检索都不能自动升级成 Memory。它们最多是 Memory 系统里的某些组件，而不是系统本身。

### 生命周期：写入、整理、读取

- **写入**：决定什么值得记。关键不是存得下，而是别把噪声、猜测、瞬时状态和真正有价值的信息混成一锅粥。
- **整理**：负责合并去重、冲突调解、版本替换、衰减归档、标记失效。不会整理的系统，积累的往往不是智慧而是误解。
- **读取**：不是机械回放过去，而是围绕当前任务临时组装一个“可行动的过去版本”，通常涉及混合召回、重排序、过滤和预算裁剪。

从这个角度看，Memory 不是保存过去，而是持续重解释过去。

### 原始材料与派生材料

严肃的 Memory 系统需要同时持有两层材料：

- **Raw**：原始会话、工具轨迹、观察记录，保真但分散、昂贵、难直接使用。
- **Derived**：摘要、画像、偏好标签、关系图谱，紧凑可用，但会在多轮改写中产生漂移。

如果只有 Raw，系统会钝；如果只有 Derived，系统会漂。高质量架构要求二者之间存在可回溯、可校验、可重构的路径。

### 遗忘与治理

Forget 不是 Delete。删除一条原始记录，不等于删除由它派生出的摘要、偏好、策略和后续行为影响链。成熟的遗忘机制必须带有版本意识、时间意识和依赖传播意识。

继续往下一层，Memory 的核心问题也不只是技术实现，而是治理：

- 这条记忆来自原话、摘要还是推断？
- 新旧信息冲突时谁覆盖谁？
- 用户要求删除时哪些派生物必须连带失效？
- 哪些敏感记忆不能跨场景带出？

所以 Memory 更像操作系统，而不只是数据库。

### 记忆的终点是技能

当经验被反思、提炼、反复验证，并沉淀为可复用行为结构时，Memory 就从“保存过去”走向了“塑造未来行为”。这也是很多 Agent 系统把 Skills 当作程序性记忆层的原因。

从这个意义上说，经验最开始只是“发生过”，之后变成“总结过”，最后才真正变成“会做了”。
