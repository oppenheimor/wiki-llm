# Claude Identity Verification

> Anthropic 在 Claude 平台上逐步推出的身份验证流程。用户在访问部分能力、触发平台完整性检查或其他安全合规措施时，可能需要提交政府签发的实体身份证件并完成自拍验证。

## 核心要点

- **触发不是全量常驻**：官方表述是“针对少数使用场景逐步推出”，会在访问特定能力、例行平台完整性检查或其他安全与合规措施时弹出验证。
- **验证材料要求严格**：需要实体的政府签发带照片证件、带摄像头设备，以及几分钟时间；不接受截图、扫描件、电子身份证或非政府证件。
- **Anthropic + Persona 分工**：Anthropic 是数据控制方，Persona Identities 是验证处理方；证件照片和自拍照由 Persona 持有，不直接存到 Anthropic 系统。
- **数据用途被明确收窄**：Anthropic 官方明确说验证数据不会用于模型训练，不会超范围收集，也不会用于营销或广告等与验证无关的用途。
- **访问门槛开始显性化**：这件事意味着“是否能用到更强能力”不再只由价格决定，也受到平台审核、地区支持和身份合规约束影响。

## 详细说明

Anthropic 在 2026-04-14 更新的帮助中心文章中正式公布了 Claude 平台身份验证机制。官方给出的动机有三类：防止滥用、执行使用政策、履行法律义务。

从流程看，这套机制并不等同于“所有用户一注册就强制实名认证”。更准确的说法是：Anthropic 已经把身份验证变成一个可按需触发的安全闸门，在特定能力入口、安全风控检查或其他合规场景里启用。

官方要求用户准备：
- 实体的政府签发带照片证件
- 带摄像头的手机或电脑
- 一次通常少于五分钟的验证流程

可接受证件包括护照、驾照、州/省级身份证、国民身份证等。不可接受的包括复印件、截图、扫描件、电子/移动版证件、学生证、工牌和临时纸质证件。

## 与 OpenAI 组织验证的对照

OpenAI 已经在 API 平台推出 `Organization Verification`。两者共同点是都把“高级能力访问”与政府证件验证绑定，并都将目标表述为减少滥用与保持更安全的能力开放。

差异在于：
- **Anthropic**：面向 Claude 产品平台，按 use case 逐步触发，覆盖“访问某些能力”与“平台完整性检查”
- **OpenAI**：面向 API 组织级访问控制，用于解锁更多高级模型和图像能力，并限制一张证件 90 天内只能验证一个组织

这说明头部模型厂商正在把“身份验证”从例外措施，推进为先进能力分发机制的一部分。
