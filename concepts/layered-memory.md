# Layered Memory（分层记忆体系）

> Hermes Agent 将记忆组织为三层（加一层技能）持久化系统，实现提示词稳定与历史可检索之间的平衡。

## 三层记忆结构

### 第一层：核心持久记忆（MEMORY.md / USER.md）

- 经过精心整理的智能体笔记和用户偏好，约 **1.3k tokens**
- 每次会话开始时加载进提示词，会话期间保持不变
- 优先级最高、始终可用，但容量严格受限
- 记忆更新不影响当前会话，只在新会话或重建时生效
- 内存刷新机制：长对话压缩前，模型有机会将重要长期信息提取到这两文件

### 第二层：会话历史（SQLite + FTS5）

- 容量更大、无限扩展的历史对话可检索记忆
- 存储在 `~/.hermes/state.db`
- 通过 **FTS5** 实现全文索引
- 智能体按需精准提取过往上下文片段，通过 LLM 摘要重构后纳入推理

### 第三层：Honcho 长期画像（可选）

- 帮助智能体建立用户的**长期画像**
- 记住偏好、习惯和行为模式，而非单次会话细节

### 第四层（隐含）：程序化技能层

- 技能作为独立"程序性记忆"存在
- 存储的不是智能体"知道什么"，而是"学会做什么"
- 即跨会话的可复用工作流和行动方式

## 架构图

![Hermes 分层记忆架构](图片来源：Hermes Agent 官方文档)

三层结构如下：

| 层次 | 名称 | 容量 | 存储位置 | 行为 |
|------|------|------|----------|------|
| 第一层 | 核心持久记忆 | ~1.3k tokens | MEMORY.md / USER.md | 会话开始时加载，全程不变 |
| 第二层 | 会话历史 | 无限制 | `~/.hermes/state.db` | 按需检索，FTS5 全文索引，压缩后存储 |
| 第三层 | Honcho（可选） | — | — | 用户长期画像 |
| 第四层 | Skills | — | `~/.hermes/skills/` | 程序化记忆，存储"会做什么" |

核心设计原则：**常驻内存小巧且精心筛选，其他一切都当作可检索历史。**

内存栈设计目标：**令牌效率**和**按需检索**。常驻内存小巧且精心筛选，其他一切都当作可检索历史。

## 与 OpenClaw 记忆对比

| 维度 | Hermes | OpenClaw |
|------|--------|----------|
| 存储介质 | SQLite + Markdown | Markdown 文件 |
| 更新方式 | 动态、可随时更新 | 显式、文件权威 |
| 压缩策略 | LLM 摘要 + 内存刷新 | 人工整理 |
| 可检索性 | FTS5 全文索引 | 文件系统搜索 |
