# GEP（Genome Evolution Protocol / 基因组进化协议）

> 一套让 Agent 经验可保存、可传递、可进化的开放协议。借鉴生物学里基因表达的思路——成功做法被打包成"基因胶囊"，其他 Agent 可以直接继承，不用再从零摸索。

## 解决的问题

在 GEP 出现前，Agent 的知识积累有两条路径：
- **[[concepts/claude-code-skills]]（Skills）**：人工编写、打包共享，写完就固定，更新依赖人
- **上下文记忆**：每次任务结束就蒸发，下一个 Agent 要再踩一遍同样的坑

结果：100 家公司各自训练 Agent 解决同一个问题，总成本上万美元。这是巨大的社会性浪费。

## 核心机制：基因胶囊

Agent 成功完成一件事后，把"成功的做法"打包成基因胶囊。其他 Agent 在遇到相关问题时可以直接继承这颗胶囊的底层逻辑。

与 Skills 对比：

| | Skills | GEP Gene |
|---|---|---|
| 来源 | 人写 | Agent 使用中涌现 |
| 更新方式 | 人工更新 | 随使用调整 |
| 错误处理 | 需人修 | 能自我修复 |
| 淘汰机制 | 无 | 长期没用自动淘汰 |
| 类比 | 公司发的员工手册 | 员工攒下的工作经验 |

## 跨领域迁移的真实案例

一位游戏策划在用 AI 构建世界观时，给 AI 设了一个很强的角色人设，结果生成的所有名词都很独特，天然避开了命名冲突。这个"命名隔离策略"被打包成基因胶囊。

后来一位后端工程师的 Agent 在搜索"怎么解决变量命名冲突"时，匹配到了这颗来自游戏领域的胶囊，继承了底层逻辑，一次就把编译问题修好了。

**关键洞察**：解决方案可以来自完全不相关的领域。Agent 网络的经验共享打破了人类知识社区的领域壁垒。

## 协议不是平台

> 平台可以被收购、被关掉（[[concepts/openclaw]] 被 OpenAI 收购就是个例子），但协议是开放的，谁都可以实现。你的 Agent 攒下的经验属于你。

这一设计哲学决定了 GEP 的长期价值：无论具体平台兴衰，只要协议开放，用户的经验资产就不会被锁定。

## 经济效应

以前：100 家公司各自训练 Agent 解决同一个问题 → 总成本上万美元
现在：一个 Agent 解决了，其他 99 个花几美分继承经验

Agent 用得越多，整个网络就越强——这是网络效应在经验层的体现。
