# ChatBot、Copilot 与 Agent 对比

> 三者的本质差异不在模型强弱，而在“谁掌控循环”：人驱动、人主导 AI 建议、还是 AI 自主推进任务。

## 对比维度

| 对比维度 | ChatBot | Copilot | Agent |
|----------|---------|---------|-------|
| 控制权 | 人掌控循环 | 人掌控循环，AI 有建议权 | AI 掌控循环，人保留否决或审批权 |
| 交互粒度 | 一问一答 | 上下文感知后给建议 | 基于目标持续多轮执行 |
| 自主性 | 几乎没有 | 有局部主动建议 | 有任务级自主规划与行动 |
| 是否主动调工具 | 通常不会 | 很少，且由人确认 | 会，且会基于结果继续决定下一步 |
| 典型场景 | 问答、写作、解释 | 代码补全、局部改写、IDE 建议 | 重构、迁移、调试、跨系统操作 |
| 用户角色 | 提问者 | 驾驶员 | 副驾 / 审批者 |

## 各自优势

### ChatBot
- 成本低、反馈快，适合问答、解释、草稿生成
- 用户完全掌控节奏，出错边界清晰

### Copilot
- 在不打断主工作流的前提下提供局部增益
- 适合用户仍想亲自驾驶，但希望 AI 帮忙补全和建议

### Agent
- 适合“给目标，不给步骤”的任务
- 能把读文件、搜信息、改代码、跑命令、看结果串成闭环
- 在复杂任务上比“一轮一轮手动推动”更高效

## 选型指南
选择 ChatBot，如果任务主要是问答、解释、头脑风暴，且不需要它真正操作环境。

选择 Copilot，如果你已经在某个工作流里，想要一个随时给建议的搭档，但最终控制权仍必须牢牢在你手里。

选择 Agent，如果任务的关键难点不只是“生成文本”，而是“连续判断下一步并操作外部世界”，例如代码重构、测试修复、信息收集后执行动作。
