# Hermes Agent 与 OpenClaw 对比

> OpenClaw 赋予用户完全掌控权，Hermes Agent 让智能体通过使用不断自我进化——两者代表了"个人智能体"这一概念的两种不同范式。

## 核心差异

OpenClaw 通过赋予用户强大的控制权、结构化管理和本地身份，使自托管智能体切实可用。Hermes 走向另一条路：把智能体视为一个能力可以不断积累、而非仅仅执行任务的系统。

## 架构对比

| 对比维度 | Hermes Agent      | OpenClaw           |
| ---- | ----------------- | ------------------ |
| 核心思路 | 智能体执行循环为核心        | 中心化 Gateway 控制器为核心 |
| 进化方式 | 自我进化（执行—学习—改进闭环）  | 人工塑造（用户直接掌控）       |
| 设计哲学 | "你的智能体会被工作流塑造和优化" | "你的智能体，完全由你掌控"     |

## 技术细节对比

| 对比维度 | Hermes Agent | OpenClaw |
|----------|-------------|----------|
| 内存机制 | 分层持久化（SQLite+FTS5） | Markdown 文件（事实权威来源） |
| 技能来源 | 自动生成 + 人类优化 | 主要由人类编写 |
| 记忆更新 | 动态、可随时更新 | 显式、文件化、可版本管理 |
| 个性锚定 | 全局（SOUL.md 在 HERMES_HOME） | 工作区绑定（SOUL.md 在工作区目录） |
| 定时任务 | 内置 cron 系统，开箱即用 | 原生支持定期回顾上下文（主动提醒模式） |
| 安全性默认 | 五层纵深防御，默认更安全 | 以操作者为核心，加固依赖用户 |
| 模型依赖 | 无限制，任何 OpenAI API 兼容端点 | 无限制 |
| 安装体验 | 几乎开箱即用，一条命令安装 | 需要用户手动配置 |
| 基础设施 | 研究型，开发者友好 | 有 NVIDIA、OpenAI 加持，更完善 |
| 平台支持 | 不支持原生 Windows（需 WSL2） | 支持多平台 |

## 各自优势

### Hermes Agent
- 自我进化潜力大，长期使用能力不断积累
- 分层记忆机制更易于维护
- 开箱即用，配置门槛低
- 内置五重安全机制，默认更安全
- 适合开发者、研究者

### OpenClaw
- 多平台消息接口更成熟
- 文件驱动身份系统可版本控制
- 可深度定制和加固
- 以操作者为核心的完全透明
- 适合更广泛的用户群体

## 选型指南

**选择 Hermes Agent**，如果：
- 需要自托管长期运行的 AI 助手
- 希望智能体随使用不断进化
- 需要定时执行任务并输出结果
- 希望个性在各场景下保持一致
- 重视内置安全默认配置

**选择 OpenClaw**，如果：
- 需要最强大的个人助理控制台
- 希望助理真正融入你的数字环境
- 重视操作透明性和可审计性
- 需要多平台消息集成
- 希望深度定制和版本化管理

## 互补建议

**不要局限于单一选择**。两个智能体可以并行运行，输出结果可复用，甚至可通过简单接口实现互联。两者结合可以兼具用户直接掌控和能力复利。
