# Agent 7×24 在线宿主机

> 个人 Agent 想从“偶尔打开的软件”升级为“持续在线的助理”，前提不是模型更强，而是背后要有一台稳定在线、不断电、不掉网的宿主机。

## 问题场景

很多本地或自托管 Agent 框架都支持定时任务、主动提醒、远程调用、消息通道或手机遥控。但只要它们依赖的那台机器关机、睡眠、断网或无人值守，这些能力就会瞬间退化成“只有你坐在电脑前时才可用”。

这类问题在以下场景尤其明显：
- 需要 Agent 定期检查监控、邮箱、消息或任务队列
- 希望用手机远程给 Agent 下指令
- 希望出差、睡觉、离开办公桌时 Agent 仍继续工作
- 希望把“主动工作”变成常态，而非临时演示

## 解决方案

把 Agent 的运行前提拆成两层：

1. **Agent 软件层**：Hermes、OpenClaw、Claude Code + Routines、Nexus4CC 等各自解决记忆、工具调用、调度或远程入口问题
2. **宿主机基础设施层**：一台 7×24 在线的机器，负责持续供电、联网、持久运行和远程可达

可选宿主机形态包括：
- 本地常开设备：Mac mini、NUC、小主机、家用服务器
- 远程主机：VPS、云服务器、托管机
- 混合模式：本地开发 + 云端常驻执行

真正要点不是“买哪家机器”，而是先承认：**Agent 的持续可用性是基础设施问题，不是提示词问题。**

## 权衡取舍

- **本地常开**
  - 优点：数据在手边，开发和调试最直接
  - 缺点：停电、断网、睡眠、外出无人处理时最脆弱

- **云端宿主机**
  - 优点：供电、联网、可达性更稳定，更适合长期在线与远程调用
  - 缺点：需要额外运维、成本、权限隔离和数据放置判断

- **工作电脑混跑**
  - 优点：省一台机器
  - 缺点：最容易把 Agent 运行环境和日常开发环境搅在一起，出现依赖污染、权限风险或系统稳定性问题

## 与云端运行的区别

“有一台 7×24 在线宿主机”和“Agent 自己在厂商云端运行”不是一回事。

- [[concepts/claude-code-routines]]：云端运行，Anthropic 托管执行环境，本地机器不是硬前提
- [[concepts/hermes-agent]] / [[concepts/openclaw]]：更常见的是用户自持宿主机，自行保证在线性

这两类方案都能解决“人不在电脑前”时的可用性问题，但责任边界不同：
- 厂商云：平台负责基础设施，用户负责任务与权限边界
- 自托管宿主机：用户同时负责 Agent 本身与宿主机稳定性
