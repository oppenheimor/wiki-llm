# SkillForge

> 阿里云提出的垂直领域 Agent Skills 自进化框架，把企业技能包的生成、部署、诊断与优化串成自动闭环。

## 核心能力
- **领域情境化生成**：从历史工单、内部文档和任务描述中抽取工作流、工具与知识，生成初始 Skills。
- **运行期性能监控**：将 Agent 输出与专家参考结果比对，把失败案例回流到后续诊断流程。
- **多维度故障分析**：从知识、工具、澄清、风格四个维度并行识别失败原因。
- **自动诊断与优化**：把故障模式映射到具体 skill 片段，再最小化修改 `SKILL.md` 或参考资料生成新版本。
- **安全约束结构**：通过去脚本化 skill 包与 VFS 交互边界，降低企业生产环境中的执行风险。

## 技术亮点
### 它把 Skill 生命周期从“人工写说明书”升级为“可观测优化闭环”
SkillForge 关注的不只是生成第一版 skill，而是 skill 上线之后如何持续进化。核心不是一次性 prompt 产物，而是失败反馈、诊断报告和自动修订之间的闭环。

### 企业级 Skill 强调“领域扎根”而非通用泛化
其设计假设是：企业场景的高质量技能，必须从真实流程、真实知识和真实工具中长出来，而不是靠通用模型即兴生成。因此它更像“领域工程化 skill 工厂”，而不是通用 skill 生成器。

## 使用场景
- 云技术支持、IT 运维、内部服务台、电商客服等垂直企业场景
- 需要持续迭代 Agent Skills、又不希望完全依赖人工维护的团队
- 关心 skill 安全边界、可追溯性和版本演化的生产系统
