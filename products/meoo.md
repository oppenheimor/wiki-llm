# Meoo（秒悟）

> 阿里推出的 AI 应用生成产品，主打“思维未竟之处，秒悟已然成真”，通过自然语言和预置工作流快速生成网页与应用原型。

## 核心能力

- **自然语言生成应用**：用户直接描述要做什么，产品负责生成对应的前后端应用形态，目标用户不局限于传统开发者。
- **预置案例覆盖多类场景**：官网首页展示了文件处理、官网门户、创意游戏、效率工具、AI 应用、社区论坛、创意 H5 等多类模板与案例。
- **文件驱动的应用生成**：官方首页预置了 CSV、Excel、PPT、PDF 简历等输入案例，说明它不只做纯文本对话，也支持“带素材建应用”的路径。
- **技能化工作流入口**：体验文章显示，Meoo 把 Skill 作为可复用工作流单元，支持调用现成能力、上传自定义 Skill，并沉淀经验。
- **可视化修改与协作**：体验文章提到，它支持框选界面元素再用自然语言下达修改指令，也支持多人同时参与编辑。

## 技术亮点

### 官方首页本身就暴露了“应用工坊”定位
`meoo.com` 的标题是 “Meoo秒悟”，描述语是“思维未竟之处，秒悟已然成真”。首页脚本里的 `window.MEOO_CONFIG.homepage_showcase` 直接列出了大量生成目标和 prompt 模板，说明它的产品心智不是“聊天助手”，而是“把输入编译成应用”。

### 文件与网页两条入口同时成立
官方预置案例既包含 SaaS 官网、作品集、餐厅官网，也包含图表、Excel 展示页、PPT 展示页、简历转个人网站。这意味着它把“从零生成”和“从现有素材派生”都作为一等路径。

### 体验稿把它描述成中文语境下的 Lovable 类产品
微信公众号《阿里下场做了中国版的 Lovable，好用到炸。》把 Meoo 明确类比为 “阿里的 Lovable”，并强调其蜂群式多智能体协同、与阿里云能力打通、Skill 复用和长上下文压缩体验。这里更适合作为用户体验视角，而不是官方定义。

## 使用场景

- 非开发者或轻技术用户把脑中的点子快速做成可运行网页
- 以文件为输入，生成数据展示页、在线演示页、个人主页等成品
- 通过语音或自然语言反复迭代 UI 和交互，进行 Vibecoding 式快速原型
- 结合团队协作，把一个想法从演示原型推进到带后端能力的应用雏形
